# Code of Conduct

## Purpose

WorkGrid Memory should be a practical, respectful, and technically serious project community.

The project welcomes contributors who want to improve local-first AI context, developer tooling, privacy-preserving memory systems, and MCP-based workflows.

## Expected Behavior

Participants are expected to:

- Be respectful and constructive
- Discuss ideas without attacking people
- Give useful technical feedback
- Assume good intent when reasonable
- Keep disagreements focused on the work
- Respect privacy and sensitive data
- Avoid sharing private user data, credentials, or personal profile information
- Report security issues responsibly

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment or personal attacks
- Threats or intimidation
- Doxxing or sharing private information
- Publishing secrets, private profile data, or private workspace content
- Deliberate disruption of discussions or pull requests
- Spam
- Abuse of project issue trackers
- Discriminatory language or behavior
- Sexualized comments or inappropriate attention

## Technical Discussions

Strong opinions are allowed. Bad behavior is not.

It is acceptable to say:

```text
This design may leak sensitive profile fields through MCP. We need field-level filtering.
```

It is not acceptable to say:

```text
Only an idiot would design it this way.
```

Sadly, the second one is often easier to type. Resist the urge. Civilization limps forward.

## Privacy-Sensitive Contributions

Because WorkGrid Memory may handle personal profiles, workspace content, images, and private workflows, contributors must be extra careful when posting examples.

Do not post:

- Real secrets
- Real private profile data
- Private photos
- Customer data
- Private source code without permission
- Logs containing personal data

Use fake data or sanitized fixtures.

## Enforcement

Project maintainers may:

- Edit or delete inappropriate comments
- Close issues or pull requests
- Temporarily restrict participation
- Permanently ban abusive participants
- Report severe abuse to GitHub
